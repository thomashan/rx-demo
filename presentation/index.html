Why reactive?

It turns out many companies working on different domains were independently discovering patterns for building software that look the same. Reactive!

Reactive programming provides a higher abstraction than imperative or declarative programming, ultimately increasing productivity in domains where RX exceled at.

Analogy: productivity gained by employing java over assembly as a general programming language.

Why reactive?

The reactive extensions (RX) originally developed by Microsoft started all.

Application using RX showed the properties outlined in reactive manifesto.

What is reactive, reactive programming, RX recapped

Reactive: mindset that follows the reactive manifesto

Reactive Programming: programming paradigm that works with streams of data

RX: library that works with observables (streams of data)

Other libraries which implements reactive programming are Akka actor and Akka stream not part of the RX family.

Let's make Service Engines more reactive

Add more resilience

1% chance of any node going down independently

99%

The probability of six endpoints going down is 1%

Or the uptime will be 99%

Now lets divide the end points so they fail independently

99.99%

The probability of six endpoints going down is 0.01%

Or the uptime will be 99.99%

We've gained more resiliency by setting the services up so they fail independently.

The initial cost maybe higher but maintenance cost will far outweigh the initial costs.

Global State

Set data-state="something" on a slide and "something" will be added as a class to the document element when the slide is open. This lets you apply broader style changes, like switching the page background.

State Events

Additionally custom events can be triggered on a per slide basis by binding to the data-state name.
